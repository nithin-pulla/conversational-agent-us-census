# US Census Conversational Agent

A natural-language agent that answers real questions about US population, demographics, income, housing, and education — by generating SQL and executing it live against the [US Open Census Data](https://app.snowflake.com/marketplace/listing/GZSTZ491VXY) dataset on Snowflake.

Ask *"Which state has the highest median household income?"* and get back a number that matches the Census Bureau's published figure, not an average-of-averages that's $10,000 off.

---

## How it works

```
User question
      │
      ▼
┌─────────────┐   off-topic   ┌──────────────────────┐
│  Guardrail  │ ────────────► │  Rejection message   │
└─────────────┘               └──────────────────────┘
      │ on-topic
      ▼
┌──────────────────────────────────────────┐
│  Hybrid Retriever (schema context)       │
│  ├── BM25 (sparse / exact keyword)       │
│  ├── Neural Dense (all-MiniLM-L6-v2)    │
│  └── Reciprocal Rank Fusion (RRF)        │
└──────────────────────────────────────────┘
      │ top-k schema entries
      ▼
┌──────────────────────────────────────────┐
│  ReAct Agent Loop (up to 5 steps)        │
│  ├── THOUGHT → SQL → validate → run      │
│  ├── Inspect results, refine query       │
│  └── FINAL_ANSWER                        │
└──────────────────────────────────────────┘
      │
      ▼
  Streamlit UI
```

| File | Responsibility |
|---|---|
| `agent.py` | Guardrail, ReAct reasoning loop, final synthesis |
| `retrieval.py` | Hybrid Neural Dense + BM25 schema retriever |
| `db.py` | Snowflake connection, SQL safety validation, query execution, schema inspector |
| `app.py` | Streamlit chat UI |

---

## Why the retrieval is hybrid

The Census Bureau's column naming convention is not designed for human readability. `B19013e1` is median household income. `B25003e2` is owner-occupied housing units. Pure keyword search fails the moment a user's phrasing diverges from the government's terminology.

The retriever runs two passes and fuses them:

- **BM25 (sparse):** Fast exact-keyword match. Works great for column codes (`B19013`) and known terms (`income`, `population`).
- **Neural Dense (`all-MiniLM-L6-v2`):** Sentence-transformer embeddings that bridge the vocabulary gap. `"home ownership"` retrieves `"owner occupied tenure"`. `"commute time"` retrieves `"travel time to work"`. `"poverty rate"` retrieves `"below poverty level"`.
- **Reciprocal Rank Fusion:** Both ranked lists are merged using RRF (`k=60`) — no score normalisation needed, no magic weights to tune.

A Census-specific synonym dictionary (`retrieval.py`) expands queries before retrieval as an additional bridge for the most common mismatches.

If `sentence-transformers` is unavailable, the retriever falls back to BM25-only mode without crashing.

---

## Why median income is tricky

`B19013e1` is a pre-computed median at the Census Block Group level. Averaging it across CBGs does not give the true county or state median — the distribution within each CBG is discarded, and the result can be off by $10,000 or more.

The agent reconstructs the correct median by aggregating income bracket counts (`B19001e2`–`B19001e17`) across all CBGs and interpolating. The result matches the Census Bureau's published state figures to within ~$50.

## Security model

Two layers of protection sit between the LLM and the database:

**1. SQL validation in code (`db.py`)**

Every query generated by the agent is validated before it reaches Snowflake:
- The first real keyword must be `SELECT` or `WITH` (after stripping `--` and `/* */` comments).
- Every semicolon-separated segment is scanned for blocked verbs: `DROP`, `DELETE`, `INSERT`, `UPDATE`, `CREATE`, `ALTER`, `TRUNCATE`, `GRANT`, `REVOKE`, `COPY`, `CALL`, `EXECUTE`, `USE`, and more.
- Violations raise `UnsafeQueryError` before any connection is opened — no warehouse credits consumed, no round-trip to Snowflake.

**2. Least-privilege database role**

The application connects as a custom `CENSUS_READER` role, not `ACCOUNTADMIN`. That role has exactly two permissions: `USAGE` on the warehouse and `SELECT` on the census dataset. Even if the SQL validator had a gap, Snowflake would refuse the write at the database level.

See [Setting up the read-only role](#setting-up-the-read-only-role) below.

## Local development

### 1. Clone

```bash
git clone https://github.com/nithin-pulla/conversational-agent-us-census.git
cd conversational-agent-us-census
```

### 2. Virtual environment

```bash
python3.11 -m venv .venv
source .venv/bin/activate        # macOS / Linux
.venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

On first run, `sentence-transformers` downloads `all-MiniLM-L6-v2` (~90 MB). Cached locally after that.

> Running tests? Use the dev requirements instead:
> ```bash
> pip install -r requirements-dev.txt
> ```

### 4. Configure credentials

```bash
cp .env.example .env
```

Open `.env` and fill in:

```env
SNOWFLAKE_ACCOUNT=your_account_identifier   # format: orgname-accountname
SNOWFLAKE_USER=your_username
SNOWFLAKE_PASSWORD=your_password
SNOWFLAKE_DATABASE=US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET
SNOWFLAKE_SCHEMA=PUBLIC
SNOWFLAKE_WAREHOUSE=COMPUTE_WH
SNOWFLAKE_ROLE=CENSUS_READER                # see setup below — do not use ACCOUNTADMIN

OPENROUTER_API_KEY=your_openrouter_api_key

LOG_LEVEL=INFO
```

### 5. Setting up the read-only role

Run the following once in Snowflake as `ACCOUNTADMIN`, substituting your app username:

```sql
CREATE ROLE IF NOT EXISTS CENSUS_READER;

GRANT USAGE ON WAREHOUSE COMPUTE_WH TO ROLE CENSUS_READER;

GRANT USAGE ON DATABASE US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET TO ROLE CENSUS_READER;
GRANT USAGE ON SCHEMA US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET.PUBLIC TO ROLE CENSUS_READER;
GRANT USAGE ON SCHEMA US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET.INFORMATION_SCHEMA TO ROLE CENSUS_READER;

GRANT SELECT ON ALL TABLES IN SCHEMA US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET.PUBLIC TO ROLE CENSUS_READER;
GRANT SELECT ON FUTURE TABLES IN SCHEMA US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET.PUBLIC TO ROLE CENSUS_READER;

GRANT ROLE CENSUS_READER TO USER your_username;
```

The `INFORMATION_SCHEMA` grant is required because the app queries it on startup to build the schema retrieval index.

### 6. Verify connectivity

```bash
python - <<'EOF'
from db import run_query
rows = run_query('SELECT CURRENT_USER(), CURRENT_ROLE(), CURRENT_DATABASE()', max_rows=1)
print(rows)
EOF
```

Expected: your username, `CENSUS_READER`, and the census database name.

### 7. Run the tests

Unit tests (no Snowflake or network required):

```bash
pytest tests/ -m "not integration" -v
```

Integration tests (requires a live Snowflake connection):

```bash
pytest tests/ -m integration -v
```

### 8. Start the app

```bash
streamlit run app.py
```

Opens at `http://localhost:8501`. First load fetches schema metadata from Snowflake and builds the hybrid retrieval index (15–30 seconds). Every question after that is served from the cached in-memory index.

## Deployment

### Option A — Streamlit Community Cloud (recommended, free)

1. Push the repo to GitHub.
2. Go to [share.streamlit.io](https://share.streamlit.io), sign in with GitHub, click **New app**.
3. Set repository, branch (`master`), and main file (`app.py`).
4. In **Advanced settings**, set Python version to `3.11`.
5. In the **Secrets** panel, paste:

   ```toml
   SNOWFLAKE_ACCOUNT   = "orgname-accountname"
   SNOWFLAKE_USER      = "your_username"
   SNOWFLAKE_PASSWORD  = "your_password"
   SNOWFLAKE_DATABASE  = "US_OPEN_CENSUS_DATA__NEIGHBORHOOD_INSIGHTS__FREE_DATASET"
   SNOWFLAKE_SCHEMA    = "PUBLIC"
   SNOWFLAKE_WAREHOUSE = "COMPUTE_WH"
   SNOWFLAKE_ROLE      = "CENSUS_READER"
   OPENROUTER_API_KEY  = "sk-or-v1-..."
   ```

6. Click **Deploy**. First boot takes 2–3 minutes to install dependencies and download the embedding model.

## Troubleshooting

**`KeyError: SNOWFLAKE_ACCOUNT`**
Credentials are not set. Locally: check `.env` exists and is filled in. On Streamlit Cloud: verify secrets are pasted correctly in the Secrets panel.

**`OperationalError: 250001` (connection refused)**
`SNOWFLAKE_ACCOUNT` must be in `orgname-accountname` format, not a full URL.

**`Insufficient privileges` when connecting as `CENSUS_READER`**
The role setup SQL hasn't been run, or was run while connected as a different admin. Re-run the role setup as `ACCOUNTADMIN` and confirm with `SHOW GRANTS TO ROLE CENSUS_READER`.

**`OPENROUTER_API_KEY` rate limit / 429**
The agent retries up to 3 times with exponential backoff. If errors persist, check your OpenRouter usage dashboard.

**Slow first response**
The first question triggers schema fetch + index build + model inference. Subsequent questions are fast (2–5 s) because the index is cached in memory by Streamlit's `@st.cache_resource`.

**`sentence-transformers` not found**
Run `pip install -r requirements.txt`. The retriever falls back to BM25-only mode without crashing.

**Docker build fails on `torch`**
`torch` is a transitive dependency of `sentence-transformers` (~2 GB). Increase Docker's memory limit or build on a machine with at least 4 GB available to Docker.

**`No matching distribution found for streamlit`**
Upgrade pip first: `python -m pip install --upgrade pip`, then retry.
